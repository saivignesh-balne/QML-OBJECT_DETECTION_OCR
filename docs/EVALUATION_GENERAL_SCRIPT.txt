INTERNSHIP EVALUATION GENERAL PRESENTATION SCRIPT
Project: Research on Hybrid Quantum Machine Learning for Object Classification and OCR

Good morning.

Today I am presenting my internship project, which is focused on researching Quantum Machine Learning, or QML, in a practical computer-vision setting. The core idea of my work was not only to build a working object detection and OCR system, but to use that system as a realistic platform to compare classical machine-learning approaches with hybrid quantum approaches.

The application context I selected was packaging analysis. In this project, the system is designed to work with three object classes: bottles, chip packets, and medicine boxes. Given an input image, the pipeline first detects the object, then extracts the relevant region of interest, classifies the object, and finally extracts any printed text visible on it using OCR.

The reason I chose this problem is because it is practical and technically rich at the same time. Packaging images are challenging because they include cluttered backgrounds, reflections, different lighting conditions, and text that may be small or partially visible. This makes it a meaningful real-world problem for experimentation. At the same time, it creates a strong environment for QML research, because it allows us to test whether hybrid quantum models can contribute useful classification performance in a non-trivial setting.

The main objective of my internship was therefore to study how QML can be integrated into a realistic AI pipeline and to evaluate whether it provides measurable value compared with strong classical baselines. So, the packaging pipeline is the application layer, but the deeper internship objective is the research comparison between classical and quantum-oriented models.

The system architecture begins with OpenCV-based preprocessing. This stage handles denoising, contrast enhancement, and binarization. These preprocessing steps are important because both object detection and OCR are sensitive to image quality. After preprocessing, the full image is passed to the detector, which in my implementation is YOLO. The detector identifies the object and gives bounding boxes. From those bounding boxes, the relevant region of interest is extracted.

Once the ROI is obtained, it is passed into two branches. The first branch is the classification branch. Here, I use classical feature extraction methods to convert the ROI into a compact feature representation. On top of those features, I compare different classifier families. These include classical classifiers such as Classical SVM, Logistic Regression, Random Forest, and MLP, as well as quantum-oriented classifiers such as QSVM with different kernels and encodings, and variational quantum classifiers.

The second branch is the OCR branch. For text extraction, I used both Tesseract OCR and TrOCR. The system uses an ensemble-style selection approach so that if one OCR backend performs poorly on a given image, the other can still provide a valid output. If text is not readable, the system still returns the detected object label and marks the extracted text as No label.

An important design decision in this project was to use a hybrid quantum approach rather than attempting a pure quantum solution. A pure quantum pipeline would try to process image data directly in quantum form, but that is not practical in this context because images are high-dimensional and current near-term quantum systems cannot handle that type of input efficiently. So, the practical and research-relevant choice was hybrid quantum machine learning. In this design, preprocessing and feature extraction remain classical, while the final classifier stage becomes the place where QML is introduced and evaluated.

Within the QML part of the project, I implemented both angle encoding and amplitude encoding. Angle encoding maps classical features to rotation angles in a quantum circuit. Amplitude encoding prepares the classical data as a normalized quantum state. On top of these encodings, I evaluated quantum-kernel support vector machines and variational quantum classifiers. This gave me a way to study not only whether QML can work, but also which style of quantum modeling is more suitable in this task.

To make the project more complete and usable, I also built a full user interface using Flask and React. The UI allows training from the browser, benchmark generation, and upload-based inference. More importantly, it turns the project into a repeatable evaluation environment. The UI stores trained models, shows benchmark charts and comparison tables, and displays final full-image results with extracted text. This made the project more presentation-ready and also helped me structure the research outputs more clearly.

Coming to the internship goals and objectives, this project helped me in multiple ways. In terms of application, I applied machine learning, computer vision, OCR, and QML concepts in a complete working pipeline. In terms of knowledge, I learned how QML can fit into a realistic system instead of existing only as a theoretical model. In terms of exploration, I experimented with multiple model families, kernels, and encodings. In terms of identification, I identified where quantum methods appear promising and where classical methods still remain stronger.

For innovation, I built a benchmark-driven hybrid classical-quantum platform instead of only training a single model. For engagement, I worked across multiple technical layers, including datasets, model training, benchmarking, OCR, UI design, and documentation. For evaluation, I compared all the models on the same held-out dataset and measured them using the same logic. For demonstration, I created a project that can be shown not only as code, but as a polished system with reports, documentation, and visual outputs.

Now coming to the results, the detector achieved strong performance. The current detector metrics include precision of 0.90054, recall of 0.94974, mAP at 50 of 0.95593, and mAP at 50 to 95 of 0.61159. On the ROI classification side, the best classical models achieved 99.5 percent held-out accuracy. Interestingly, one quantum-oriented model, the amplitude-encoding QSVM, also achieved 99.5 percent accuracy on the same ROI dataset.

This is an important result for the internship because it shows that QML can be competitive in this task. However, the deeper comparison also shows that competitive accuracy alone is not enough to conclude that QML is better. The amplitude-encoding QSVM required significantly more training time than the best classical models. Also, when we look at the full family-level results, the average classical performance is still stronger and more stable than the average quantum-family performance.

So the most honest and technically sound conclusion of this project is the following: for immediate production use, the best choice is still the classical stack, especially YOLO combined with a classical SVM and OCR ensemble. But from a research and innovation perspective, hybrid QML is highly valuable, because it demonstrates that quantum-oriented methods can be integrated into a real pipeline and, in at least one case, can match the top classical accuracy.

That means the project delivers two outcomes at the same time. First, it delivers a practical deployment path through the classical benchmark winner. Second, it delivers a meaningful innovation path through the hybrid QML experiments. In my view, this dual outcome is one of the strongest aspects of the project, because it balances practical engineering with research value.

In terms of future scope, the next important step would be to carry out a more complete end-to-end validation on full-scene test images instead of relying only on proxy composite benchmarking. Another important area is improving OCR robustness on difficult package labels. On the QML side, future work could include testing richer quantum kernels, alternative encoding strategies, and better feature-compression methods before quantum processing. The main principle I would follow is to continue QML experimentation only where it provides clear and measurable value over strong classical baselines.

To conclude, this internship helped me understand not only how to build a hybrid QML application, but also how to evaluate QML responsibly in a practical setting. The project shows that classical models are currently the strongest production option, while hybrid QML remains the strongest direction for research and innovation. So, the real contribution of this internship is not simply building a packaging-recognition system, but building a benchmarked framework to understand the role of QML in a realistic AI pipeline.

Thank you.
