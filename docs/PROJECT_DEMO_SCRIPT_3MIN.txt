FULL DEMO SCRIPT
--------------------------------------------------

Hello everyone.

In this demo, I will show my internship project, which is a hybrid quantum machine learning pipeline for object detection, classification, and OCR.

The main purpose of this project is to compare classical and quantum-oriented models in a practical object-detection workflow. The current benchmark uses chip packets, medicine boxes, and bottles as experimental object classes.

This is the main dashboard of the application. The UI is built to support the complete workflow, including model training, benchmarking, and inference. It also stores trained models so that we do not need to retrain them every time.

First, in the training section, I can train the detector and the ROI classifiers. The detector is responsible for finding the object in the image, and the classifier is responsible for identifying whether the object is a chip packet, medicine box, or bottle. I can also train multiple models for comparison, including classical models and hybrid quantum models.

Next, in the benchmark section, the system compares all trained models and shows their performance using graphs and tables. This is one of the important parts of the project, because the goal is not just to build one working model, but to evaluate how classical methods and quantum-based methods perform under the same pipeline.

Now I will show the inference flow.

Here in the Inference Lab, I upload a new image. Once the image is uploaded, the system processes it through several stages. It first performs preprocessing such as noise reduction, contrast enhancement, and binarization. Then the detector identifies the object region. After that, the ROI is passed to the classifier, and the OCR module extracts any visible printed text.

The output is shown directly on the image. Here, we can see the predicted object label, and we can also see the extracted text recognized from the package. If text is not clearly visible, the system still returns the object prediction and marks the text output accordingly.

Another useful part of this project is that the inference lab can compare multiple models on the same image. That means I can evaluate how a classical model behaves compared with a hybrid quantum model, and then use the benchmark section to decide which approach is best for production and which is more useful for research.

From the current results, the classical models are the strongest and most stable for immediate deployment. However, the hybrid quantum models are important in this project because they show how QML can be integrated into a realistic application and compared fairly against strong classical baselines.

So this project is not only a demo application. It is also a research and benchmarking platform. It helps answer an important question: where does quantum machine learning provide value, and where do classical models still remain the better choice?

To conclude, this demo shows a complete end-to-end system that combines object detection, ROI classification, OCR, model benchmarking, and a user-friendly interface. The final outcome of the project is both practical and research-oriented: classical models are the current production winners, while hybrid quantum models provide the main innovation and experimentation value.

Thank you.
